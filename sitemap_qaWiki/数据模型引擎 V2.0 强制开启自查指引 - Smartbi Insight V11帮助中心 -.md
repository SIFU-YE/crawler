
# 1 背景
为提升系统性能与功能体验，2025-08-05 后的版本包将强制开启数据模型引擎 V2.0。若您此前使用过 V1.0，更新 war 包前请务必完成以下自查，避免影响业务。​
# 2 自查步骤
1、若项目版本为 2025 年 4 月之前，且在运维设置 / 系统选项 / 高级设置中 OLAP_QUERY_NEW_ENGINE_BUILD_MDX 设为 false 或未设置，则全局将采用数据模型V1.0；如果要直接升级到最新版本，建议先在测试环境更新，或者立即联系我们，技术团队将协助评估升级方案。
2、若缓存库非以下类型，系统也会默认采用数据引擎 V1.0，如需更新版本，可联系我们：

3、若项目版本为 2025 年 4 月之后的版本，可以用以下 SQL 语句在产品内置"SYSTEM知识库"中查询哪些数据模型使用了 V1.0 ：
SELECT c_id FROM `tx_augmented_dataset` where c_define like '%"useSmartCube":false%'
