Presto 是一款**分布式 SQL 查询引擎** ，专为高效查询大规模分布式数据而设计。它的核心定位是**跨数据源的交互式分析引擎** ，适用于大数据分析、数据探索、实时报表等场景。 涉及到数据写入方面有诸多限制，因此，涉及到写操作的功能都无法支持。  
注意：“sql_require_primary_key”参数必须要设置为OFF，取消强制要求所有新建表必须包含主键（Primary Key） 数据连接使用MySQL驱动传入的catalog参数无效，导致无法添加非默认的catalog的表（直接写SQL语句没有这个限制），因此，产品也无法支持该数据库的catalog使用。  
Trino（原名 PrestoSQL）是一个**分布式 SQL 查询引擎** ，专为高效查询大规模分布式数据而设计，其核心定位是 “**在数据所在的地方查询数据** ”，适用于大数据分析、数据湖查询、交互式分析等场景。 涉及到数据写入方面有诸多限制，因此，涉及到写操作的功能都无法支持。  
2、数据库本身不支持catalog（例如：在数据库中只能执行 Select * from schema.table，执行 Select * from scatalog.schema.table 就会报错），因此，产品也无法支持该数据库的catalog使用。 3、数据库存储过程本身不支持返回游标结果（只能返回整型值），因此，产品也无法支持该数据库的存储过程。
