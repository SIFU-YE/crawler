
# 1、概述
用户使用透视分析过程中，发现不能满足业务数据需求，需要查询清单明细数据，新建一个即席查询，还需要把所有的字段及条件等重新选择一遍，非常的麻烦，希望能自己转为即席查询。
# 2、功能说明
## 2.1 功能入口
即席查询工具栏 -> 更多 -> 进入即席查询模式。
## 2.2 转换规则说明
**字段转换基本原则：**
当前报表使用的，目标报表也能使用的的所有字段都传递到目标报表中，包括行区、列区、度量区、筛选条件区的所有字段信息
  * 字段：用户的模型中的所有字段、私有字段（分组字段、计算列）
  * 字段内属性：显示名、排序、数据格式
  * 筛选条件区的字段及字段内属性：控件类型、联动过滤条件、排序方式
  * 参数：参数的基本设置信息


  * “行”、“列”、“度量区”的字段转到即席查询“表格/列字段”区
    * 字段的维内排序全部转换全局排序
  * “过滤条件”的字段转到即席查询“过滤条件”中


透视分析转即席查询，无法转换的内容包括：（注：只能透视分析使用的字段信息不能带过去）
  * 度量的汇总依据不转
  * 命名集、私有命名集不转
  * 计算度量、私有计算度量不转
  * 当透视分析存在不能转的字段时，点击转换后，弹出提示信息：“转换为即席查询后，表格上的快速计算、计算度量和命名集，将自动被清空，是否继续？”
